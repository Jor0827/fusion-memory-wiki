# Contributing to Fusion Memory Wiki

> A lightweight contribution guide for this documentation and methodology project.

---

## What This Project Is

This is **documentation and methodology**, not code:
- Markdown files explaining the fusion memory architecture
- Wiki pages with knowledge synthesis
- Usage guides and setup instructions

**You don't need to know programming to contribute.**

---

## How to Contribute

### 1. Improve Documentation

Found a typo? Confusing explanation? Please fix it.

```markdown
# Just edit the file and submit a PR
```

### 2. Add Wiki Pages

Have knowledge worth sharing? Create a new page:

```
wiki/pages/
├── entities/      # Projects, tools, people
├── concepts/       # Methods, theories
├── decisions/      # Why we chose X
└── synthesis/      # Comprehensive analysis
```

Follow the template in [wiki/SCHEMA.md](wiki/SCHEMA.md).

### 3. Share Use Cases

- Reply in [Discussions](https://github.com/qclaw/fusion-memory-wiki/discussions)
- Share how you're using the system
- Report what works and what doesn't

### 4. Report Issues

- Bug reports: Open an Issue with label `bug`
- Feature requests: Open an Issue with label `enhancement`
- Questions: Use [Discussions](https://github.com/qclaw/fusion-memory-wiki/discussions)

---

## Pull Request Guidelines

### Small Changes

For typos, small fixes, or clarifications:
1. Edit directly on GitHub (click the pencil icon)
2. Submit PR with a brief description

### Larger Changes

For new wiki pages or major rewrites:
1. Fork the repository
2. Create a branch: `git checkout -b docs/new-topic`
3. Make changes
4. Submit PR

### PR Description Template

```markdown
## What Changed
Brief description of your changes.

## Why
Why are these changes valuable?

## Screenshots (if applicable)
Add screenshots or before/after comparisons.
```

---

## Quality Guidelines

### Do ✅

- Be clear and concise
- Include examples when possible
- Link to related pages
- Update [index.md](wiki/index.md) when adding pages

### Don't ❌

- Add duplicate pages (check index.md first)
- Use complex jargon without explanation
- Leave placeholder text like `[TODO]`

---

## Questions?

Feel free to open a Discussion or Issue. No contribution is too small.
